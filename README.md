# Multi-Agent ML Pipeline

Мультиагентная система на базе LangGraph, которая автономно решает задачу машинного обучения: от сбора данных до сравнительной оценки моделей и формирования отчёта.

## Бизнес-задача

Заказчик - интернет-магазин, которому необходим легковесный автоклассификатор отзывов для снижения времени реакции на проблемы с товарами.

Бизнес-задача проекта - снизить время реакции интернет-магазина на проблемы с товарами за счёт автоматического выявления негативных клиентских отзывов.

Магазин получает большой поток клиентских отзывов, и ручной анализ всей обратной связи занимает много времени и ресурсов. Из-за этого проблемы с товарами или сервисом обнаруживаются слишком поздно, когда они уже влияют на удовлетворённость клиентов и продажи.

Разрабатываемый ИИ-агент строит автоклассификатор, который автоматически разбивает входящие отзывы на позитивные и негативные. Это позволяет бизнесу приоритизировать обработку жалоб, быстрее находить проблемные товары, улучшать карточки товаров, корректировать ассортимент и повышать удовлетворённость клиентов. 

Технически задача решается как бинарная классификация текста: модель получает на вход текст отзыва и предсказывает метку - 1 или 0, а также вероятность негативного класса. На выходе бизнес получает список отзывов с метками и вероятностями: отзывы с высокой вероятностью негативного класса автоматически попадают в очередь на разбор команде поддержки.

В качестве обучающих данных агент самостоятельно находит публичный датасет отзывов (например, Amazon Product Reviews) через встроенные библиотеки Python или открытые источники в интернете. Разметка берётся напрямую из датасета, который уже содержит колонку Positive, сформированную на основе звёздного рейтинга покупателей: 4-5 звёзд -> 1 (позитивный), 1-2 звезды -> 0 (негативный). Агент не размечает данные вручную.


| Агент | Модель | Роль |
|---|---|---|
| orchestrator | claude-sonnet-4-6 | Планирование пайплайна и финальный отчёт |
| data_collector | claude-sonnet-4-6 | Сбор датасета |
| data_preprocessor | gemini-3.1-pro-preview | Очистка и feature engineering |
| data_validator | claude-sonnet-4-6 | Проверка качества данных |
| data_analyzer | gemini-3.1-pro-preview | EDA и определение целевой переменной |
| trainer | claude-sonnet-4-6 | Обучение и сравнение ML-моделей |
| model_reviser | claude-opus-4-7-think | Оценка качества модели |
| summarizer | gpt-5.4-nano | Markdown-резюме сессии |


## Используемые библиотеки

| Библиотека | Назначение |
|---|---|
| `langgraph` | Построение мультиагентного графа (StateGraph, условная маршрутизация, retry-циклы) |
| `langchain-openai` | LLM-клиент (ChatOpenAI) для всех трёх провайдеров через OpenAI-совместимый API |
| `python-dotenv` | Загрузка API-ключей из `.env` файла |
| `tavily-python` | Поиск датасетов в интернете (используется агентом data_collector как последний резерв) |
| `pandas` | Загрузка, обработка и сохранение табличных данных (CSV) |
| `scikit-learn` | ML-модели (LogisticRegression, RandomForest, LinearSVC), TF-IDF, GridSearchCV, метрики |
| `seaborn` | Доступ к встроенным датасетам (titanic, diamonds и др.) для быстрого сбора данных |

## Как работает система

Пользователь вводит задачу на естественном языке -> система создаёт сессию с уникальным timestamp и запускает граф агентов.

**1. Orchestrator (планирование)**
Получает задачу, смотрит историю прошлых сессий из `long_term_memory.json`, решает какие агенты нужны и в каком порядке. Возвращает `plan: [список агентов]`.

**2. Data Collector**
Ищет подходящий датасет: сначала sklearn/seaborn built-ins, потом прямые URL, в крайнем случае - поиск через Tavily. Сохраняет `raw/dataset.csv`.

**3. Data Preprocessor**
Чистит данные (дубли, пропуски, выбросы), создаёт новые признаки (feature engineering). Сохраняет `processed/clean.csv`.

**4. Data Validator**
Проверяет качество: ≥5000 строк, наличие текстовых и категориальных колонок. При провале - автоматический **retry**: preprocessor запускается заново с пометкой об ошибке.

**5. Data Analyzer**
EDA: корреляции, распределения, определяет целевую переменную и тип задачи (classification/regression/forecasting). Сохраняет `reports/analysis.json`.

**6. Trainer**
Сравнивает 2–3 модели, подбирает гиперпараметры через GridSearchCV на train (80%). Сохраняет `models/model.pkl` и `processed/test.csv` (held-out 20%).

**7. Model Reviser**
Оценивает модель **только на test.csv** (данные, которых модель не видела). При плохих метриках - **retry**: возвращает trainer или весь цикл сбора данных.

**8. Summarizer**
Читает логи всех агентов, пишет краткое `summary.md`. Добавляет запись в `long_term_memory.json` для следующих сессий.

**9. Orchestrator (отчёт)**
Получает все результаты, генерирует `final_report.md` с бизнес-выводами и `final_artifact.json` со структурированными метриками.

Агенты взаимодействуют через общий `AgentState` (LangGraph): каждый агент читает нужные поля и записывает свои результаты. Дополнительно агенты могут передавать `notes` следующему напрямую - например, валидатор пишет препроцессору что именно исправить.

## Артефакты

Каждый запуск создаёт папку `data/sessions/<timestamp>/`:

```
data/sessions/2026-04-28_00-41-13/
├── raw/dataset.csv           # сырые данные
├── processed/clean.csv       # очищенный датасет
├── models/model.pkl          # лучшая модель
├── reports/
│   ├── validation.json
│   ├── analysis.json
│   ├── evaluation.json
│   └── summary.md
├── final_report.md           # отчёт для пользователя
├── final_artifact.json       # структурированные результаты
└── conversations.json        # полный лог агентов
```

Долгосрочная память хранится в `data/long_term_memory.json` - при каждом новом запуске оркестратор видит историю предыдущих сессий.

## Техники промптинга

| Техника | Агенты |
|---|---|
| Role-based prompting | все агенты |
| Contrastive prompting (Bad/Good) | data_preprocessor, trainer, model_reviser |
| Chain-of-thought | data_analyzer, model_reviser, orchestrator |
| Dual-mode prompting | orchestrator (planning vs responding) |
| Extended thinking | model_reviser (claude-opus-4-7-think) |
| Inter-agent notes | data_validator -> data_preprocessor, model_reviser -> trainer |
